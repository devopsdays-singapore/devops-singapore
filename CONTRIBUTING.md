# Contributing to DevOps Singapore

Thank you for your interest in contributing to DevOps Singapore!
This community is powered by people like you — speakers, organizers, volunteers, and attendees.

### Preferred Ways to Contribute

1. **Open a GitHub Issue** — This is the most effective channel (inspired by SingaporeJS).
2. Comment on existing event issues.
3. Submit Pull Requests for documentation or template improvements.

### Specific Contribution Types

**Proposing a Talk**
- Use the [CFP template](https://github.com/devopsdays-singapore/devops-singapore/issues/new?template=cfp.md)
- Or comment directly on an upcoming event issue with your idea

**Event Feedback / Suggestions**
- Comment on the relevant event issue (open during planning)
- All past events are archived as closed issues

**Helping Organize**
- Venue ideas, AV support, food, photography, sponsorship outreach
- Open an issue labeled `help-wanted`

**Jobs or Sponsorships**
- Open a new issue with the appropriate label (`jobs` or `sponsor`)

### Communication Channels

- **GitHub Issues** → Best for structured discussion and permanent record
- **Telegram** → For quick chat and announcements (links in README)
- **Luma** → For RSVPs and event reminders

### Pull Requests

We welcome small improvements to templates, README, or assets.  
Please keep changes focused and add a clear description.

### Code of Conduct

All contributors and attendees must follow our [Code of Conduct](CODE_OF_CONDUCT.md). Be respectful and kind.

---

Questions? Just open an issue — we're happy to help!
